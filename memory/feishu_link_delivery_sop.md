# Feishu Link Delivery SOP

## Problem
Feishu (Lark) message surfaces often fail to render Markdown links `[Title](URL)` as clickable elements, especially for cloud documents. This leads to user friction where they have to copy-paste URLs manually.

## Solution
Always deliver critical URLs (like Feishu Docs, Wiki, Bitable) as **RAW text** on a dedicated line.

## Core Rule
**DO NOT use:** `[Click here](https://...)`
**ALWAYS use:** `https://...`

## Memory Entry (Updated 2026-03-02)
- [P0] All Feishu document links generated by JARVIS must be sent as raw URLs.
- [SOP] When a tool returns a `doc_url`, the assistant must extract the URL and present it clearly without Markdown link wrapping.
